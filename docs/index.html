iOS Coding Challenge: Fancy Filter

Build a nice Objective C API for the Fancy Filter C++ library.

Motivation

At Adobe, there is a huge corpus of C++ code, especially for doing image processing. A common task is to take these C++ libraries and provide Objective C APIs that can be called from iOS apps. The goal is to provide a clean, iOS-style API that makes the filter as intuitive as possible to incorporate.

Challenge

  • Download the latest Fancyfilter release. You will find a README.md, fancyfilter.hpp header file, and a FAT static library called libff.a which provides symbols for the iphoneos and iphonesimulator platforms.
  • Refer to the Fancyfilter documentation.

Considerations

C++ implementation black box. Because you cannot modify the underlying C++ filter implementation. Release build -O3 so this is maximum performance. no debug symbols. Library doesn't provide any checking for data integrity, etc. You can decide on if you want to include the library as a submodule, or embed directly in your source... or have a build step to download it, etc. Whatever you do, having some type of unit tests or app harness for verifying the correctness of your API will be key. When done, please submit your solution (new fancy filter API, test functions, test app, etc), to

Deliverable

Zipped Xcode project directory or a link to github repo that contains the Xcode project. It can be either an iOS or MacOS project.
