iOS Coding Challenge: Fancy Filter

Build a clean Objective C API for the Fancy Filter C++ library.

Motivation

It is very common for high-performance image processing code to be written in C/C++. Including these C/C++ APIs in iOS apps is not always straightforward, so it is not uncommon to to take these C++ libraries and provide Objective C wrapper APIs that can be called from iOS apps.

Challenge

The goal is to provide a clean, iOS-style API for the Fancy Filter C++ library that makes the filter as intuitive as possible for iOS developers who may not be familiar with C/C++ to incorporate. It will be up to you to decide things like:

  • What the API should look like
  • How to test your API (XCTest, test app, etc)
  • How you would distribute the result (static library, framework, source code, etc)
  • How you want to include the contents of the Fancy Filter repo (submodule, build-time download, etc)
This challenge is less about your ability to write extremely complex iOS code, and more about API design, iOS best practices, and your ability to work, at least superficially, with C/C++ code.

Getting Started

  • Take a look at the Fancy Filter repo
  • Clone or download the latest Fancy Filter release. The important files are: README.md, fancyfilter.hpp header file, and a FAT static library called libff.a which provides symbols for both the iphoneos and iphonesimulator platforms.
  • README.md should provide all of the necessary information for getting a new Xcode project building with the library.

Deliverable

Zipped Xcode project directory or a link to github repo that contains the Xcode project. It should be an iOS project. The final project should not just contain your new API, it should also provide an executable (XCTest, app, etc) that can be used to verify the correctness of the implementation.

Rules

  • You can use any available resource you can find on the Internet — eg, Stack Overflow, API docs, etc.
  • You may not ask questions related to the problem on Stack Overflow, IRC, etc. (eg, the Internet is "read-only")
  • You cannot use any 3rd party libraries or products, you can only use classes and API’s in the standard iOS/Cocoa/Objective-C runtime. If in doubt, just ask.
